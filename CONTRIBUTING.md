# Contributing (kursprojekt)

## Branch-namn
- `feature/<kort-namn>`
- `fix/<kort-namn>`
- `docs/<kort-namn>`

Exempel: `feature/weekly-report`, `fix/import-bug`

## Issue → PR
1) Skapa/ta en issue.
2) Skriv kort *problem*, *acceptanskriterier*, *testidé*.
3) Skapa branch från `main`.
4) Gör små commits med tydliga meddelanden.
5) Öppna PR tidigt (gärna “draft”) och begär review.

## PR-regler (rekommenderat)
- Minst 1 review innan merge.
- CI måste vara grön.
- PR ska länka till issue: “Closes #<id>”.
- Om PR är stor: skriv en “plan” i beskrivningen.
